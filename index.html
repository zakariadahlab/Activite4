Partie 01 : Résume de chapitre 1

  1)  Les techniques de l’information et de la communication (TIC)

  Les TIC (Techniques de l'information et de la communication) regroupent des outils pour manipuler, produire, transmettre et échanger des informations sous différentes formes. Elles sont nées de la convergence de l'informatique, des télécommunications, de l'électronique et de l'audiovisuel. Les utilisateurs peuvent communiquer, accéder à des informations, stocker, manipuler et produire des données à travers des techniques comme l'informatique, l'audiovisuel, le multimédia et les télécommunications.

  2)  Catégories et objectifs

  Les TIC se divisent en 3 catégories : la téléinformatique pour l'échange d'informations numériques, les télécommunications pour les systèmes communicants non informatiques, et l'électronique grand public pour les équipements tels que les télévisions et lecteurs de DVD. Les objectifs des TIC sont la connectivité, la modularité, l'implantation simple, l'utilisation facile et la fiabilité pour la détection et correction des erreurs.

  3)  Composants de TIC

  Matériel : appareils tels que PC, télévision, téléphone portable, tablette, appareil photo numérique, robots, etc. ; station de travail capable d'envoyer des données vers les réseaux. Logiciel : systèmes d'exploitation, logiciels d'application et d'utilisation des différents composants matériels ; réseaux d'internet ou mobiles.

  4)  Caractéristiques

  La communication numérique offre des avantages tels que la rapidité d'exécution des activités, l'accès amélioré à l'information pour tout le personnel, la rationalisation de la communication pour les informations confidentielles, la mondialisation des activités, la répartition et l'intensité de l'information circulant entre les différentes stations.

  5)  Initiation aux techniques de communication

  Objectifs de communication: Les objectifs de communication correspondent aux réactions attendues du récepteur. Ils sont définis selon trois dimensions: cognitive, affective et comportementale.
  Système de communication: Un système de communication comprend des partenaires de communication (émetteur et récepteur), des vecteurs (message et médias), des fonctions (codage, décodage, réponse, feedback) et le bruit qui peut perturber les signaux.




  Types de communication: Il y a deux types de communication: institutionnelle (communication d'image de marque) et commerciale ou de produit (mise en avant des caractéristiques et avantages du produit).
  Schémas de communication: Trois schémas sont employés: la communication interpersonnelle, la communication de groupe et la communication de masse.




  Média et Multimédia: Les médias (presse, radio, télévision, internet) sont des moyens impersonnels de diffusion d'informations, tandis que le multimédia permet l'utilisation interactive de plusieurs modes de représentation de l'information.

   Internet et Web

   Comprendre les mécanismes fondamentaux du Web: URL, HTTP et HTML

 La leçon porte sur le Web et ses aspects techniques. Le Web est un système hypertexte public sur Internet qui permet de consulter des pages accessibles sur des sites. Il est relié par des hyperliens qui relient les pages web entre elles. Le Web n’est qu’une des applications d’Internet. Les trois mécanismes de base du Web sont l'URL, HTTP et HTML. L'URL est le nom unique de la page décrivant l'emplacement d'une ressource. Le HTTP est un protocole de niveau applicatif qui suit le paradigme client-serveur. Le HTML est un langage de description de pages Web qui décrit chaque élément de la page. Une page Web est stockée dans un fichier HTML et interprétée et affichée par un navigateur. Un site Web est un regroupement de pages Web autour d'une thématique commune reliées entre elles par des liens hypertextes. Pour créer une page Web, on utilise le langage HTML, Les documents HTML sont créés à l'aide de balises et doivent comporter les balises HTML, HEAD, TITLE et BODY. Les titres et paragraphes peuvent être ajoutés à l'aide des balises H et P. Les documents HTML sont des fichiers texte structurés et interprétés dans un navigateur web.

  Git :

  Git est un logiciel de versioning créé en 2005 par Linus Torvalds. Il permet de conserver un historique des modifications effectuées sur un projet comme toutes les autres logiciels de gestion de versions qui sont devenues indispensables pour tout dévloppeur. Ils aident les groupes travaillants sur un projet de developpement à diviser les taches et sauvegarder leurs progrès pour mieux de synchronisation en premettant à plusieur version d'un seul projet d'exister en cas d'erreur ou juste pour corrdonner entre les membres.





La façon dont Git considère les données est différente des autres systèmes, il les voit comme des fluxs d’instantanés, chaqe instantané est stocké dans une base de données locale quand en enregistre notre projet cela rend les opérations beaucoup plus rapides localement sans le besoin d'être connecté à un serveur externel. Pour suivre ces modfications Git divise les fichiers en deux catégories:

  • Non suivi: les fichiers sont au départ tous non suivis quand on importe un répertoire déjà existant
  • Suivi: il appartenait au dernier instantané capturé par Git.cet état est lui même composé de trois états:
    • Modifié (“modified”):quand on ajoute des nouveaux fichiers ou on les modifie Git va les considérer modifié.
    • Indexé (“staged”):lorsqu’on indique à Git que le fichier modifié doit faire partie du prochain instantané,donc tant qu'il ne sont pas dans l'état "modifié".
    • Validé (“committed”):lorsque le fichier est stocké dans la base de données locale.Lors du clonage d’un dépôt déjà existant il est validé par défaut.

   Zones de travail


  Ces états de fichiers sont liés à des zones de travail dans Git :


  • Le répertoire de travail ou “working tree” correspond à une extraction unique d’une version du projet.
  • La zone d’index ou “staging area” correspond à un simple fichier qui stocke les informations du prochain instantané ou du prochain “commit”.

  Clonage de dépôt Git

 Lorsqu’on démarre avec Git nous pouvons importer l’ensemble des ressources d’un projet stockés localement sur serveur distant pour implémenter un système de gestion de version,la deuxième façon de démarrer un dépôt Git est de cloner localement un dépôt Git déjà existant. Ca nous permet après d'ajouter ou de modifier des fichiers dans notre projet en Git utilisant plusieurs autres commandes (git add,git commit...) et pour actualiser notre dépôt plus la possibilité de consulter l'historique des modifications utilisant "git log", par conséquent ouvrir un des principaux intérêts d’utiliser un logiciel de gestion de vision: l'annulation des modifications en cas d'erreur ou changement d'intérêt avec "git restore".
